**Care Team section (3 handlers):**

1. `handle_create_care_team(args: dict) -> dict` -- Required: `patient_id`, `name` (team name, e.g. "ICU Care Team - Patient Smith"). Optional: `encounter_id`, `status` (default "active", enum: proposed/active/suspended/inactive/entered-in-error), `participants` (list of dicts, each with `role_code` (SNOMED), `role_display`, `member_display`). Builds CareTeam per FHIR R4: resourceType, status, name, subject ref, encounter ref, participant array where each has role (coding with SNOMED system http://snomed.info/sct) and member display. Routes through approval queue with `ActionType.CARE_TEAM`. Returns care_team id, name, participant_count, action_id, message.

2. `handle_get_care_team(args: dict) -> dict` -- Read-only. Required: `patient_id`. Optional: `encounter_id`, `status` (default "active"). Searches CareTeam with patient=, optional encounter=, status=. Returns total + list (id, name, status, participants extracted with role + member display).

3. `handle_update_care_team_member(args: dict) -> dict` -- Read-modify-write. Required: `care_team_id`, `action` (enum: "add"/"remove"), `member_display` (name). Optional: `role_code` (SNOMED), `role_display`. If action="add", appends participant. If action="remove", removes participant matching member_display. Catches 404. Returns care_team_id, action performed, member, updated participant count, message.

**Goals section (3 handlers):**

4. `handle_create_goal(args: dict) -> dict` -- Required: `patient_id`, `description` (text, e.g. "MAP > 65 mmHg without vasopressors"). Optional: `encounter_id`, `lifecycle_status` (default "active", enum: proposed/planned/accepted/active/on-hold/completed/cancelled/entered-in-error/rejected), `achievement_status` (default "in-progress", enum from http://terminology.hl7.org/CodeSystem/goal-achievement), `target_measure_code` (LOINC code), `target_measure_display`, `target_value` (number), `target_unit`, `start_date` (ISO date), `category` (e.g. "clinical"). Builds Goal per FHIR R4: resourceType, lifecycleStatus, achievementStatus coding, description.text, subject ref, target array if measure provided (with LOINC coding + detailQuantity), startDate, category coding. Routes through approval queue with `ActionType.GOAL`. Returns goal id, description, lifecycle_status, action_id, message.

5. `handle_get_patient_goals(args: dict) -> dict` -- Read-only. Required: `patient_id`. Optional: `lifecycle_status` (filter), `encounter_id`. Searches Goal with patient=, optional lifecycle-status=, optional target-date=. Returns total + list (id, lifecycle_status, achievement_status, description text, targets extracted).

6. `handle_update_goal_status(args: dict) -> dict` -- Read-modify-write. Required: `goal_id`, `lifecycle_status` (enum). Optional: `achievement_status` (enum). Reads Goal, updates lifecycleStatus and optionally achievementStatus. If lifecycle_status is "completed", set achievementStatus to "achieved" unless explicitly overridden. Catches 404. Returns goal_id, old_lifecycle_status, new_lifecycle_status, achievement_status, message.

**Device Metrics section (2 handlers):**

7. `handle_record_device_metric(args: dict) -> dict` -- Required: `type_code` (ISO 11073 code, e.g. "150456"), `type_display` (e.g. "SpO2"). Optional: `source_display` (device name, e.g. "Bedside Monitor - ICU Bed 3"), `category` (default "measurement", enum: measurement/setting/calculation/unspecified), `operational_status` (default "on", enum: on/off/standby/entered-in-error). Builds DeviceMetric per FHIR R4: resourceType, type coding (system urn:iso:std:iso:11073:10101), source display, category, operationalStatus. Note: DeviceMetric is device-level, NOT patient-level -- no patient_id or encounter reference. Creates via fhir_client.create. Routes through approval queue with `ActionType.DEVICE_METRIC` (use a placeholder patient_id like "device-registry" since DeviceMetric is not patient-scoped). Returns device_metric id, type, source, category, message.

8. `handle_get_device_metrics(args: dict) -> dict` -- Read-only. Optional: `type_code`, `category`, `source` (text filter). Searches DeviceMetric with optional type=, category=, source=. Returns total + list (id, type via extract_code_display, source display, category, operational_status).

**Adverse Events section (2 handlers):**

9. `handle_report_adverse_event(args: dict) -> dict` -- Required: `patient_id`, `event_description` (text). Optional: `encounter_id`, `actuality` (default "actual", enum: actual/potential), `category_code` (from http://terminology.hl7.org/CodeSystem/adverse-event-category, e.g. "medication-mishap", "product-use-error", "wrong-dose"), `seriousness` (enum: serious/non-serious), `severity` (enum: mild/moderate/severe), `date` (ISO datetime, default now). Builds AdverseEvent per FHIR R4: resourceType, actuality, category coding, subject ref, encounter ref, date, event.text, seriousness coding, severity coding. Routes through approval queue with `ActionType.ADVERSE_EVENT`. Returns adverse_event id, event_description, actuality, seriousness, action_id, message.

10. `handle_get_adverse_events(args: dict) -> dict` -- Read-only. Required: `patient_id`. Optional: `encounter_id`, `actuality`, `seriousness`, `date` (date filter). Searches AdverseEvent with patient= (uses subject param), optional encounter=, actuality=, seriousness=, date=. Returns total + list (id, actuality, date, event text, category, seriousness, severity).

**Inpatient Communication section (3 handlers):**

11. `handle_create_inpatient_communication(args: dict) -> dict` -- Required: `patient_id`, `content` (message text). Optional: `encounter_id`, `category` (default "handoff", enum: handoff/consult_request/nursing_note/interdisciplinary_note), `sender` (display name), `recipient` (display name), `priority` (default "routine", enum: routine/urgent/asap/stat), `topic` (text). Builds Communication per FHIR R4: resourceType, status="completed", category coding (using http://terminology.hl7.org/CodeSystem/communication-category system), subject ref, encounter ref, sent=now, sender display, recipient display, payload array (contentString=content), priority, topic text. Routes through approval queue with `ActionType.COMMUNICATION` (already exists in ActionType). Returns communication id, category, content preview (first 100 chars), action_id, message.

12. `handle_get_communications(args: dict) -> dict` -- Read-only. Required: `encounter_id`. Optional: `category`. Searches Communication with encounter=, optional category=. Returns total + list (id, status, category, sent, sender display, content from payload, topic).

13. `handle_search_communications(args: dict) -> dict` -- Read-only. Required: `patient_id`. Optional: `category`, `sender`, `date_from` (ISO date), `date_to` (ISO date), `encounter_id`. Searches Communication with patient=, optional filters. Returns total + list (same format as get_communications).

**IMPORTANT patterns:**
- Use module-level `fhir_client` singleton
- Write handlers route through approval queue with correct ActionType
- Read-modify-write handlers (update_care_team_member, update_goal_status) catch 404
- DeviceMetric is NOT patient-scoped (no subject reference)
- Inpatient Communication uses `ActionType.COMMUNICATION` (already exists in enum)
- Use `extract_code_display()` for CodeableConcept extraction
- Use `datetime.now(timezone.utc).isoformat()` for timestamps
