**Clinical Assessment section (4 handlers):**

1. `handle_create_clinical_impression(args: dict) -> dict` -- Required: `patient_id`, `summary` (text). Optional: `encounter_id`, `assessor` (display name, e.g. "Infectious Disease Agent"), `finding_code` (SNOMED code), `finding_display` (SNOMED display), `notes` (list of strings), `status` (default "completed", enum: in-progress/completed/entered-in-error). Builds ClinicalImpression per FHIR R4: resourceType, status, subject (Patient ref), encounter ref if provided, date (now UTC ISO), assessor display, summary, finding array (itemCodeableConcept with SNOMED coding if provided), note array. Routes through approval queue with `ActionType.CLINICAL_IMPRESSION`. Returns impression id, status, summary, action_id, message.

2. `handle_get_clinical_impressions(args: dict) -> dict` -- Read-only. Required: `patient_id`. Optional: `encounter_id`, `status` (filter). Searches ClinicalImpression with patient=, optional encounter=, optional status=. Returns total + list of impressions (id, status, date, assessor display, summary, findings extracted).

3. `handle_create_risk_assessment(args: dict) -> dict` -- Required: `patient_id`, `condition_display` (e.g. "Septic shock"). Optional: `encounter_id`, `outcome_text` (prediction outcome, e.g. "Deterioration within 6 hours"), `risk_level` (enum: negligible/low/moderate/high/certain), `basis` (list of strings, e.g. ["NEWS2 score: 9", "qSOFA score: 2"]), `notes` (list of strings), `status` (default "final", enum: registered/preliminary/final/amended). Builds RiskAssessment per FHIR R4: resourceType, status, subject ref, encounter ref, occurrenceDateTime (now), condition display, prediction array (outcome text + qualitativeRisk coding from http://terminology.hl7.org/CodeSystem/risk-probability), basis array (display strings), note array. Routes through approval queue with `ActionType.RISK_ASSESSMENT`. Returns assessment id, condition, risk_level, action_id, message.

4. `handle_get_risk_assessments(args: dict) -> dict` -- Read-only. Required: `patient_id`. Optional: `encounter_id`. Searches RiskAssessment with patient=, optional encounter=. Returns total + list (id, status, date, condition display, predictions extracted, basis extracted).

**Task Management section (5 handlers):**

5. `handle_create_task(args: dict) -> dict` -- Required: `patient_id`, `description` (the task text). Optional: `encounter_id`, `priority` (default "routine", enum: routine/urgent/asap/stat), `requester` (display, default "Supervisor Agent"), `owner` (display, assignee), `due_date` (ISO datetime string for restriction.period.end), `notes` (list of strings), `intent` (default "order"). Builds Task per FHIR R4: resourceType, status="requested", intent, priority, code.text=description, for=Patient ref, encounter ref, requester display, owner display if provided, restriction.period.end if due_date, note array. Routes through approval queue with `ActionType.TASK`. Returns task id, status, priority, description, action_id, message.

6. `handle_assign_task(args: dict) -> dict` -- Read-modify-write. Required: `task_id`, `owner` (display name of assignee). Reads Task, sets owner.display, changes status to "accepted" if currently "requested". Catches 404. Returns task_id, owner, status, message.

7. `handle_complete_task(args: dict) -> dict` -- Read-modify-write. Required: `task_id`. Optional: `output_text` (completion notes). Reads Task, sets status="completed", adds executionPeriod.end=now, adds output note if provided. Catches 404. Returns task_id, status, completed_at, message.

8. `handle_get_pending_tasks(args: dict) -> dict` -- Read-only. Required: `patient_id`. Optional: `encounter_id`, `status` (default searches for "requested,accepted,in-progress" -- comma-separated for FHIR search). Searches Task with patient= (using `for` param or patient param -- check Medplum support, fall back to `subject`), status=, optional encounter=. Returns total + list (id, status, priority, description from code.text, owner display, due_date from restriction, requester display).

9. `handle_update_task_status(args: dict) -> dict` -- Read-modify-write. Required: `task_id`, `status` (enum: draft/requested/received/accepted/rejected/ready/cancelled/in-progress/on-hold/failed/completed/entered-in-error). Reads Task, updates status. If status is "in-progress", sets executionPeriod.start=now. If "completed" or "failed", sets executionPeriod.end=now. Catches 404. Returns task_id, old_status, new_status, message.

**IMPORTANT patterns:**
- Use module-level `fhir_client` singleton
- Write handlers (create_clinical_impression, create_risk_assessment, create_task) route through approval queue
- Read-modify-write handlers (assign_task, complete_task, update_task_status) catch httpx.HTTPStatusError for 404
- Read-only handlers (get_clinical_impressions, get_risk_assessments, get_pending_tasks) do NOT use approval queue
- Use `datetime.now(timezone.utc).isoformat()` for timestamps
- Use `extract_code_display()` for CodeableConcept extraction in search results
