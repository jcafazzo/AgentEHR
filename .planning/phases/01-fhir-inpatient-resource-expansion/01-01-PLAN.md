Append after the last handler (`handle_get_immunization_status`). Use clear section headers with `# =====` separators.

**Encounter Inpatient Lifecycle section (5 handlers):**

1. `handle_create_inpatient_encounter(args: dict) -> dict` -- Creates Encounter with `class` = `{"system": "http://terminology.hl7.org/CodeSystem/v3-ActCode", "code": "IMP", "display": "inpatient encounter"}`. Required args: `patient_id`. Optional: `reason`, `admit_source` (default "emd"), `location` (default "General Ward"), `priority` (default "routine", enum: routine/urgent/asap/stat), `type_code` (SNOMED), `type_display`. Sets status="in-progress", period.start=now (UTC ISO). Routes through approval queue with `ActionType.ENCOUNTER`. Returns encounter id, status, message, action_id.

2. `handle_update_encounter_status(args: dict) -> dict` -- Read-modify-write pattern. Required: `encounter_id`, `status` (one of: planned, arrived, triaged, in-progress, onleave, finished, cancelled, entered-in-error). Catches `httpx.HTTPStatusError` for 404. If new status is "finished", sets `period.end` to now. Returns encounter_id, old_status, new_status, message. Does NOT use approval queue (status transitions are operational, not clinical).

3. `handle_get_encounter_timeline(args: dict) -> dict` -- Composite handler using `asyncio.gather`. Required: `encounter_id`. Parallel-fetches: Encounter read, Observation search (encounter=, _sort=-date, _count=50), Condition search (encounter=), MedicationRequest search (encounter=), Flag search (patient=, encounter=). Assembles timeline dict with: encounter info (id, status, class, period, location), observations (formatted: id, code via extract_code_display, value, effective_date), conditions (id, code, clinical_status, onset), medications (id, medication text, status, authored_on), flags (id, status, code text, category). Uses `return_exceptions=True` on gather -- if any sub-fetch fails, return empty list for that section with a warning. Returns the assembled timeline.

4. `handle_transfer_patient(args: dict) -> dict` -- Read-modify-write. Required: `encounter_id`, `new_location` (display string). Optional: `new_location_status` (default "active"). Reads current Encounter, appends to `location` array: new entry with display=new_location, status=new_location_status, period.start=now. Sets previous location's status to "completed" and period.end=now. Returns encounter_id, old_location, new_location, message.

5. `handle_discharge_patient(args: dict) -> dict` -- Read-modify-write. Required: `encounter_id`. Optional: `discharge_disposition` (code string, default "home"), `discharge_display` (default "Discharge to home"). Reads Encounter, sets status="finished", period.end=now, hospitalization.dischargeDisposition to coded value. Returns encounter_id, status, discharge_disposition, length_of_stay (calculated from period if possible), message.

**Clinical Flags section (3 handlers):**

6. `handle_create_flag(args: dict) -> dict` -- Required: `patient_id`, `description`. Optional: `category` (default "clinical", enum: clinical/safety/drug/lab/contact/behavioral/research/advance-directive), `encounter_id`, `priority` (enum: PN/PL/PM/PH for none/low/medium/high), `author`. Builds Flag resource per FHIR R4 spec (see research). Routes through approval queue with `ActionType.FLAG`. Returns status, message, action_id, flag dict.

7. `handle_get_active_flags(args: dict) -> dict` -- Read-only, no approval queue. Required: `patient_id`. Optional: `encounter_id`. Searches Flag with patient=, status=active, optional encounter=. Returns total count + list of flags (id, status, category via extract_code_display, code text, period).

8. `handle_resolve_flag(args: dict) -> dict` -- Read-modify-write. Required: `flag_id`. Reads Flag, sets status="inactive", sets period.end=now. Catches 404. Returns flag_id, old_status, new_status, message.

**IMPORTANT patterns to follow:**
- Use the module-level `fhir_client` singleton (never create new instances)
- All FHIR calls are async via httpx
- Format results into clean dicts (not raw FHIR bundles)
- Catch `httpx.HTTPStatusError` for 404 in read-modify-write handlers
- Use `datetime.now(timezone.utc).isoformat()` for timestamps (import from datetime)
- Use `asyncio.gather(..., return_exceptions=True)` for parallel fetches in timeline handler
- Use `extract_code_display()` helper for CodeableConcept fields
