5. `seed_inpatient_patient(profile)` -- Async function. Similar to existing `seed_patient` but handles inpatient flow: (1) find_or_create_patient, (2) build encounter resource using profile's encounter_builder(patient_id), (3) find_or_create_inpatient_encounter, (4) call profile's scenario_creator(patient_id, encounter_id) to get resources, (5) create each resource via fhir_client.create with error handling matching existing pattern.
