**1. VitalSigns dataclass:**
- Fields: `timestamp: datetime`, `heart_rate: float | None = None`, `systolic_bp: float | None = None`, `diastolic_bp: float | None = None`, `respiratory_rate: float | None = None`, `spo2: float | None = None`, `temperature_c: float | None = None`, `consciousness: str = "A"` (AVPU scale), `gcs: int | None = None`, `supplemental_o2: bool = False`, `fhir_observation_ids: list[str] = field(default_factory=list)` (provenance for NFR-04)

**2. LabResult dataclass:**
- Fields: `name: str`, `value: float`, `unit: str`, `timestamp: datetime`, `loinc_code: str = ""`, `reference_low: float | None = None`, `reference_high: float | None = None`, `is_abnormal: bool = False`, `fhir_observation_id: str = ""` (provenance for NFR-04)

**3. Finding dataclass:**
- Fields: `category: str` (e.g., "sepsis", "cardiac", "renal"), `severity: str` (maps to AlertSeverity values), `title: str`, `description: str`, `evidence: list[dict[str, Any]] = field(default_factory=list)` (citations for NFR-04), `recommended_actions: list[str] = field(default_factory=list)`, `spawn_trigger: str | None = None` (specialist agent hint for Phase 6)

**4. EvaluationResult dataclass:**
- Fields: `cycle_id: str`, `timestamp: datetime`, `cycle_duration_seconds: float`, `scores: dict[str, dict] = field(default_factory=dict)`, `alerts_generated: list[str] = field(default_factory=list)` (alert IDs), `findings: list[Finding] = field(default_factory=list)`, `spawn_triggers: list[str] = field(default_factory=list)`, `data_gaps: list[str] = field(default_factory=list)` (missing parameters)

**5. PatientState dataclass:**
- Fields:
  - `patient_id: str`
  - `encounter_id: str`
  - `patient_name: str = ""`
  - `last_updated: datetime | None = None`
  - `vitals_history: list[VitalSigns] = field(default_factory=list)` — most recent last
  - `vitals_fetched_at: datetime | None = None`
  - `lab_results: list[LabResult] = field(default_factory=list)`
  - `labs_fetched_at: datetime | None = None`
  - `active_conditions: list[dict[str, Any]] = field(default_factory=list)` — raw FHIR Condition resources (simplified)
  - `conditions_fetched_at: datetime | None = None`
  - `active_medications: list[dict[str, Any]] = field(default_factory=list)` — raw FHIR MedicationRequest resources (simplified)
  - `medications_fetched_at: datetime | None = None`
  - `care_team: list[dict[str, str]] = field(default_factory=list)` — [{name, role}]
  - `scores: dict[str, dict] = field(default_factory=dict)` — output of calculate_all_available_scores()
  - `active_alert_ids: list[str] = field(default_factory=list)`
  - `spawn_triggers: list[str] = field(default_factory=list)` — Phase 6 hooks
  - `evaluation_history: list[EvaluationResult] = field(default_factory=list)` — recent evaluations

- Methods:
  - `is_vitals_fresh(max_age_seconds: int = 60) -> bool`: Returns False if `vitals_fetched_at` is None or older than max_age_seconds from `datetime.now(timezone.utc)`
  - `is_meds_fresh(max_age_seconds: int = 300) -> bool`: Same pattern for medications_fetched_at
  - `is_conditions_fresh(max_age_seconds: int = 300) -> bool`: Same pattern for conditions_fetched_at
  - `is_labs_fresh(max_age_seconds: int = 300) -> bool`: Same pattern for labs_fetched_at
  - `latest_vitals() -> VitalSigns | None`: Returns last element of vitals_history or None
  - `get_vitals_dict() -> dict[str, Any]`: Convert latest_vitals() to the dict format expected by `calculate_all_available_scores()` — maps VitalSigns fields to the keys documented in the scoring interface (heart_rate, systolic_bp, diastolic_bp, respiratory_rate, spo2, temperature_c, consciousness, gcs, supplemental_o2). Only includes non-None values.
  - `get_labs_dict() -> dict[str, Any]`: Convert lab_results to dict keyed by standardized name (creatinine, platelets, bilirubin, lactate, etc.) using the most recent value for each. Maps to keys expected by `calculate_all_available_scores()`.
  - `to_clinical_summary() -> str`: Generate a concise text summary for LLM context. Format:
    ```
    Patient: {patient_name} (ID: {patient_id})
    Encounter: {encounter_id}

    VITAL SIGNS (as of {vitals_fetched_at}):
    HR: {hr} | BP: {sbp}/{dbp} | RR: {rr} | SpO2: {spo2}% | Temp: {temp}°C | Consciousness: {consciousness}

    ACTIVE CONDITIONS:
    - {condition display} (SNOMED: {code})
    ...

    ACTIVE MEDICATIONS:
    - {medication display} — {dosage}
    ...

    RECENT LABS:
    - {lab name}: {value} {unit} [{abnormal flag}] (LOINC: {code})
    ...

    CLINICAL SCORES:
    - NEWS2: {total_score} ({risk_level})
    - qSOFA: {score} ({sepsis_risk})
    ...

    ACTIVE ALERTS: {count}
    ```
    Uses the latest vitals only (not full history). For labs, groups by name showing most recent value. This compressed format avoids context window overflow (Pitfall 1 from research).

Create `agents/prompts/supervisor.md` with a structured system prompt for the supervisor agent's LLM calls. This prompt constrains the LLM behavior for safety (NFR-03) and evidence citation (NFR-04).

The prompt must include these sections:

**Role Definition:**
```
You are a clinical supervisor agent monitoring an inpatient. Your role is to analyze the patient's current clinical state, identify concerns, and recommend actions for clinician review.
```

**Safety Constraints (NFR-03):**
```
## Safety Rules

1. You CANNOT directly modify treatment, medications, or care plans.
2. All clinical actions MUST be recommended for clinician approval — never executed directly.
3. You CANNOT order tests, prescribe medications, or change care settings.
4. Your role is advisory: analyze, identify, recommend.
```

**Data Integrity (NFR-04):**
```
## Data Rules

1. NEVER generate, estimate, or assume clinical values that are not provided in the patient data below.
2. If a clinical value is missing, state "DATA NOT AVAILABLE" — do not infer it.
3. Every finding MUST cite specific evidence: a vital sign value, lab result, clinical score, or condition from the provided data.
4. Do not reference clinical guidelines by name unless they are provided to you. Reference the scoring systems and thresholds instead.
```

**Output Format:**
```
## Output Format

Respond with a JSON array of findings. Each finding must have:

```json
[
  {
    "category": "sepsis|cardiac|renal|pulmonary|medication|general",
    "severity": "critical|urgent|routine|informational",
    "title": "Brief finding title",
    "description": "Detailed clinical reasoning with specific data citations",
    "evidence": [
      {"type": "vital_sign|lab_result|clinical_score|condition", "name": "...", "value": "...", "timestamp": "..."}
    ],
    "recommended_actions": ["Action 1 for clinician review", "Action 2"],
    "spawn_trigger": "infectious_disease|cardiology|renal|pulmonary|medication_safety|null"
  }
]
```

If no concerns are found, respond with an empty array: `[]`
```

**Context Injection Point:**
```
## Patient Data

The following patient state will be injected before each evaluation:

{patient_clinical_summary}

## Clinical Scores (Deterministic — Pre-Calculated)

{scores_summary}

These scores are calculated from the actual vital signs and lab values above. Do not recalculate them. Interpret them and identify clinical implications.
```

**Reasoning Instructions:**
```
## Clinical Reasoning

1. Review vital sign trends — are they improving, stable, or deteriorating?
2. Cross-reference lab results with clinical scores — do they tell a consistent story?
3. Check for condition-score alignment — does the scoring match the active conditions?
4. Identify any data gaps that could affect clinical decision-making.
5. Consider medication appropriateness given current clinical state.
6. Prioritize findings by clinical urgency (critical > urgent > routine > informational).
```

The prompt uses `{patient_clinical_summary}` and `{scores_summary}` as template placeholders that the SupervisorAgent will fill with PatientState.to_clinical_summary() and formatted scores output.
